ESP8266

L’ESP8266 peut être utilisé comme un module Arduino, mais également comme un microcontrôleur à part grâce à un adaptateur USB. J’ai tout d’abord essayé, en vain, la première méthode avant d’acheter un adaptateur USB. C’est après de nombreuses heures que les premières commandes AT fonctionnèrent. J’ai dans un premier temps configuré l’ESP en serveur web en injectant une page web (l’ESP est donc connecté au routeur) pour qu’il agisse comme un serveur web. J’ai pu contrôler des LEDs et des relais grâce aux requêtes via cette page web. J’ai également essayé d’isoler l’ESP du réseau pour qu’il agisse en point d’accès wifi. En modifiant les paramètres DNS, j’ai pu contourner ceux de Google et d’Apple pour que lorsque la connexion est établie entre l’appareil et le point d’accès, la page hébergée sur le routeur s’ouvre automatiquement.

C4D : projet Arduino

Étant fasciné par les parcs d’attraction et les technologies permettant de reproduire des sensations similaires aux grandes montagnes russes (tel qu’au Futuroscope), j’ai essayé de reproduire quelque chose de similaire, nommé C4D.
3D, effets stroboscopiques, fumée, ventilateur, le tout contrôlé par un Arduino.

Première version :

Durant la première version, l’utilisateur devait scanner un code barre, préalablement donné, sur la tablette à l’entrée de l’attraction. Ce code barre permettait l’identification de l’utilisateur, récupérait le solde de tickets restant de ce dernier grâce à une application conçue exclusivement pour l’attraction. L’application décrémentait ensuite un ticket à l’utilisateur identifié avant de lui proposer une large gamme d’attractions (sous forme de mini-films) que ce dernier pouvait choisir dans une interface extrêmement intuitive pour enfin envoyer le numéro du film par Bluetooth à l’Arduino.

L’Arduino affichait alors ce numéro pour que l’opérateur puisse sélectionner l’attraction sur la télévision diffusant le film. L'opérateur n'avait alors plus qu'à lancer le film en même temps qu'il appuie sur le bouton relié au second Arduino. Ce dernier agissait sur le ventilateur grâce à un module relai s’activant lors des accélérations.

Seconde version :

Pour la seconde version, l’entièreté du projet a été amélioré. La tablette de la première version a été remplacée par un Arduino et les codes barre par des cartes RFID. La gestion du nombre de tickets a donc dû être reprogrammée au langage Arduino. La communication sans fil entre les deux Arduino a donc aussi été reprogrammée. L’utilisation du protocole Bluetooth fut conservée.

La sélection de l’attraction se faisait donc grâce à un boitier à l’entré où l’utilisateur choisissait le numéro d’attraction grâce à des boutons poussoir après avoir scanné sa carte RFID. L'Arduino envoyait donc le numéro du film au second Arduino, qui est le modèle Leonardo, ce détail a son importance car un Arduino Leonardo peut agir comme un périphérique clavier ou souris en étant connecté à un ordinateur.

L'Arduino Leonardo recevait donc le numéro de film via Bluetooth. Lorsque l’opérateur pressait le bouton poussoir commandant le lancement de l'attraction, l'Arduino Leonardo prenait le contrôle du clavier de l'ordinateur (diffusant le film sur la télé) et, grâce aux raccourcis clavier VLC, sélectionnait le bon film et le lançait. La synchronisation étant donc faite de façon autonome.

BOT Discord :

L’objectif de ce projet est d’automatiser certaines tâches en reliant des APIs telle que celle de Discord.

Son rôle est de, par exemple, accueillir les nouveaux utilisateurs en annonçant son arrivée et en remerciant la personne qui l’a invité.

Au début, le bot n’était conçu que pour un serveur, sa configuration se faisait donc via le code source. Puis le bot s’est étendu et il a fallu concevoir une interface de configuration. Cette dernière s’est faite dans un premier temps directement depuis Discord, puis s’est étendue via une page Internet.

Il a également été nécessaire de mettre en place une base de données pour stocker la configuration de chaque serveur.

Développer le serveur web m’a appris beaucoup de chose : en premier temps les configurations DNS pour relier le nom de domaine au serveur, également les chiffrements (asymétrique, symétrique…), les cookies et les requêtes.

J’ai à tout prix cherché à éviter le MySQL et le PHP, je suis donc passé par express et ejs pour rendre le site dynamique. Ce qui pour gérer les requêtes fut un véritable casse-tête.

Le programme a d’autres fonctionnalités dont l’utilisation est plus occasionnelle. Par exemple, le bot proposait des mini-jeux, des jeux de rôles…

Un sous-projet au bot fut d’organiser des tournois de jeux vidéo. Le bot lit constamment un salon textuel dans lequel l’administrateur envoie une capture d’écran des tableaux des scores. Le programme extrait les informations de l’image (pseudos et scores) et les range dans un tableur pour annoncer le vainqueur automatiquement à la fin du tournois.

Les API utilisées sont :
- MongoDB & mongoose (pour la gestion des utilisateurs et des serveurs dans la base de données)
Discord.js
- Express (pour la gestion du serveur web)
- Canvas (pour la gestion d'image notamment pour les cartes d'expérience générées à partir d'une barre de progression et de l'avatar de l'utilisateur)
- Cookies (pour stocker l'identifiant du visiteur dans son navigateur pour faciliter sa reconnexion)
- Ejs (rend la page web dynamique)
